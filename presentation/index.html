Transpiling

Converts the TypeScript code to ES5 (regular JavaScript).

Before (TS) 

function Greeter(greeting: string) {
    this.greeting = greeting;
}

Greeter.prototype.greet = function() {
    return "Hello, " + this.greeting;
}

let greeter = new Greeter("world");

let button = document.createElement('button');
button.textContent = "Say Hello";
button.onclick = function() {
    alert(greeter.greet());
};

document.body.appendChild(button);
After (JS) 

function Greeter(greeting) {
    this.greeting = greeting;
}
Greeter.prototype.greet = function () {
    return "Hello, " + this.greeting;
};
var greeter = new Greeter("world");
var button = document.createElement('button');
button.textContent = "Say Hello";
button.onclick = function () {
    alert(greeter.greet());
};
document.body.appendChild(button);

Angular2

  • 2014: announced
  • 2015: beta
  • 2016 (May): release candidate
  • 2016 (Sept): officially released
  • NOT a version update -- completely different
  • Docs for TypeScript (preferred), JavaScript, and Dart

Why Angular2?

  • Improved performance over Angular 1
  • Mobile considerations
  • Component based architecture
  • Simplification
    • Fewer ways to do the same thing
    • Less directives
    • Routing
  • Helpful documentation:
    • https://angular.io/docs/ts/latest/

Getting Started w/ angular-cli


# Install angular-cli utility globally
npm install -g angular-cli 

# Create a new project
ng new MY_PROJECT

# Build and serve the project
# Serves application at http://localhost:4200
ng serve
https://github.com/angular/angular-cli

ng new MY_PROJECT

Sets up the following:

  • Initial git configuration
  • package.json with dependencies and scripts
  • Unit tests: karma, samples, code coverage
  • e2e tests: protractor.conf.js configuration
  • Basic sample with single component

main.ts

  • Imports polyfills / environment
  • Identifies platform (platform browser)
  • Bootstraps application by importing app.module.ts

app.module.ts

Defines NgModule; bootstraps the module

Property Usage
declarations Defines the components the NgModule defines
imports NgModules that are dependencies for this module
providers Defines services or data models that will be injectable
bootstrap Name of the component to render when bootstrapping module

app.component.ts

Defines Component

Property Usage
selector Name of the HTML tag for the component
templateUrl Defines the template used when rendering the component
styleUrls Defines CSS used for this component
export class Definition of the class. Where you would hook up input/output and events.

ng generate BLUEPRINT MY_NAME

Quickly generates a new component, module, or other files. Useful for consistency.

The following items are valid "blueprints":

Component Directive
Pipe Service
Class Interface
Enum Module
